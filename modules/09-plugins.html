Plugins. Borrowed superpowers.

A plugin is a bundle someone else made: skills, subagents, hooks, integrations. Install one and Claude knows things it did not know yesterday.

What is in a plugin

Plugins ship as packages. Inside, they can include skills (module 07), subagents (module 13), hooks (module 15), and MCP servers (module 14). All registered with one install.

How to install one in Desktop

  1. Click the + button next to the message box, or run /plugins.
  2. Choose Browse marketplace. The official Anthropic registry opens.
  3. Find one. Read what it does and what permissions it asks for. Click Install.
  4. Run /reload-plugins (or restart the session) to activate.
  5. Run /help to see the new commands the plugin added.

Namespacing

Plugin skills are namespaced to avoid collisions with your own. A skill called review in a plugin called quality-tools is invoked as /quality-tools:review, not /review. This keeps your personal commands clean.

When to install, when to write your own

Install when someone has solved your exact problem better than you would. Write your own when the task is specific to how you work. Plugins are great starting points; your own skills are how you get personal leverage.

Best practices

Do

  • Stick to the official marketplace until you have a reason not to. Trust matters.
  • Read what tools the plugin requests. A note-taking plugin needing shell access is a flag.
  • Install one plugin at a time. See if it earns its place before adding another.
  • Uninstall plugins you stop using. Each one adds startup overhead.
  • Look at well-made plugins as study material. They show good skill writing.

Don't

  • Install plugins for things Claude Code already does (file edits, git, web search).
  • Pile up plugins to feel productive. Most of your power comes from your own skills and a clean CLAUDE.md.
  • Trust unsigned plugins from random repos without reading the source.
  • Forget plugins persist. If you install something and never use it, it still loads at startup.
  • Confuse plugins with MCP servers. MCPs connect Claude to remote tools (module 14). Plugins bundle Claude Code config.
Newbie trap Installing five plugins on day one to "see what they do." That is how you end up with a slow startup, prompt fatigue from unfamiliar permissions, and no idea which plugin caused which command. One at a time.

Exercise. Open /plugins. Browse the marketplace. Pick one that sounds genuinely useful for your week. Install it. Run /help and find the commands it added. Try one of them. If it does not earn its place after a week, uninstall.
