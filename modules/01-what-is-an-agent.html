What's an agent? And why it isn't ChatGPT.

Chat answers. Agents do. That single difference rearranges everything you know about working with AI.

The core idea

When you talk to Claude.ai or ChatGPT in a browser, you are talking to a very fluent text generator. It sees your message and the conversation history. That is the entire universe it has access to. It writes back. You read. You decide what to do.

Claude Code is different. Claude Code can read the files on your computer, run commands in the terminal, search the web, and edit files in place. It does this in a loop. It reads, decides, acts, checks the result, and decides what to do next, on its own, until the task is done or you stop it.

That loop is the whole game. An agent is a model with hands. Chat is a model with a microphone.

How it looks in Claude Code Desktop

You will see Claude open files on its own. You will see it run a command in a small terminal panel and read the output. You will see it edit a file and show you the diff. You can interrupt at any moment. You can roll back. You can ask it to try again differently.

The Desktop app puts all of this in one window: chat on the left, file diffs in the middle, an embedded terminal and preview when you need them. Three tabs across the top: Chat, Cowork, Code. Module 02 covers when to use which.

The mental model that helps

Stop thinking "I am asking a question." Start thinking "I am giving a job to a junior teammate who reads fast, never tires, but needs context and clear success criteria." Everything else in this guide builds on that shift.

Best practices

Do

  • Reach for an agent when the task touches files, commands, or multiple steps.
  • Give context up front: file paths, error messages, what success looks like.
  • Expect the agent to ask questions back. Answer them. That is steering, not interruption.
  • Let the agent iterate. It runs, checks, fixes. Watch the loop. Course-correct early when needed.
  • Use chat (Claude.ai or ChatGPT) for things with no side effects: explain a concept, brainstorm, summarize a paste.

Don't

  • Treat the agent like a search engine. "How does OAuth work" belongs in chat.
  • Send vague prompts and hope. "Make it faster" wastes everyone's time. State the real goal.
  • Walk away while the agent works in your real folders without checking what mode it is in.
  • Assume the agent already knows your project on the first prompt. It is reading as it goes.
  • Paste a 200-line file into chat to get help editing it. Open the project in Claude Code instead.
Newbie trap Treating Claude Code like ChatGPT-but-in-an-app. The shift is not the interface. It is that the model now has tools and a workspace. Lean into that, or you will use a Ferrari to deliver pizza one slice at a time.

Exercise. Open Claude.ai in your browser. Ask: "Rename every file in my Downloads folder that starts with 'IMG_' to start with '2026_' instead."

Watch what happens. It will explain the steps. It will not rename anything. That is chat.

Now in Claude Code Desktop, open your Downloads folder as a project and ask the same thing. Notice the difference. That is an agent.
