The slash commands you will reach for daily.

Type / in the message box and you get a menu. Most of these you will never need. About a dozen you will use every week. Here they are.

The essential dozen

  • /help — list every command and skill available right now. Always up to date. Run it when stuck.
  • /init — once per project. Generates a starter CLAUDE.md from the codebase.
  • /memory — open and edit your memory files (CLAUDE.md and auto-memory).
  • /context — see what is currently filling your context window.
  • /compact focus on … — summarize conversation while keeping the spine.
  • /clear — start a fresh session. Old session is saved to history; you can resume.
  • /resume — pick a previous session by name to keep working.
  • /model [name] — switch model mid-session. /model haiku, /model opus, /model opusplan.
  • /effort [level] — tune reasoning depth. medium, high, xhigh, max.
  • /agents — list and manage subagents (module 13).
  • /permissions — view and tweak what tools the agent can use without prompting.
  • /rename — give the current session a name so you can find it later in /resume.

Worth knowing about

  • /cost and /usage — check session and plan spend.
  • /diff — review all the edits made this session in one view.
  • /doctor — diagnose hook, MCP, or permission setup issues.
  • /loop [interval] [prompt] — run a task on a schedule. "Check the deploy every 5 minutes."
  • /plan — drop straight into Plan mode for the next request.
  • /plugins — open the plugin browser.

Custom slash commands

Anything you put in .claude/skills/<name>/SKILL.md shows up as /<name>. That is the only mechanism. (Older guides mention .claude/commands/ — still works, but skills are the modern path. See module 07.)

Best practices

Do

  • Run /help the first time you open the app. The list grows as you install plugins and skills.
  • Use /rename on any session you might want to find again. Untitled sessions are hard to spot in /resume.
  • Use /context proactively when responses get sluggish. Don't wait for compaction.
  • Keep a mental shortcut: /clear when switching tasks, /compact when continuing one.
  • Use /doctor first when something feels broken — it surfaces 80% of issues immediately.

Don't

  • Type /clear thinking you are deleting history. You are not. Old sessions stay in /resume.
  • Memorize commands you never use. /help is one keystroke.
  • Use /compact at low context. You will lose detail you still need.
  • Run /init on a project that already has a CLAUDE.md you have customized. It will overwrite.
  • Forget that arguments after the slash are a thing. /compact focus on auth works much better than bare /compact.
Newbie trap Mixing up /clear and /compact. /clear ends the session and starts a new one. /compact keeps the same session but summarizes what came before. Different tools, different moments.

Exercise. Run /help in elad-test. Read every command in the list. Then run /rename my-first-session. Close the app. Reopen. Run /resume and find your session by name. That is your save game.
