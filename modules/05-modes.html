Modes. The dial between cautious and unleashed.

A mode is how much permission you give the agent before it acts. The right mode for the moment is the difference between fast and frightening.

The five modes

Cycle through them in the Desktop app with Shift+Tab. The mode indicator sits next to the send button.

  1. Ask permissions (default). Claude asks before every file edit and every shell command. Slowest. Safest. Where you start.
  2. Auto Accept Edits. Edits and basic file operations (mkdir, mv, touch) happen automatically. Other shell commands still ask. The everyday mode for trusted work.
  3. Plan mode. Read-only. Claude can look, search, think, and propose. Cannot edit, cannot run anything. You approve the plan, then switch modes to execute.
  4. Auto. Full execution with background safety checks. Research preview. Available on higher plans. For repetitive trusted work where you want to step away.
  5. Bypass permissions. No prompts at all. Only run this inside a sandbox or VM. Never on your main machine.

The right mode at the right time

Big refactor or unfamiliar code? Plan mode first. Read the plan. Ask questions. Then switch.

You know exactly what you want and trust the change set? Auto Accept Edits. Save your wrists.

Learning, exploring, working in someone else's codebase? Ask permissions. Friction is the feature.

Sandbox VM doing nightly tasks? Bypass. Anywhere else? No.

Best practices

Do

  • Use Plan mode before any change you cannot easily undo.
  • Cycle modes mid-session with Shift+Tab. Start strict, loosen as trust grows.
  • Drop to Ask permissions when working in production code or a teammate's repo.
  • Plan mode + Opus is one of the strongest combos. Use it when stakes are high.
  • Set defaultMode: "acceptEdits" in your settings if you trust yourself in your own projects.

Don't

  • Use Bypass on your laptop. Ever. It will eventually delete something you needed.
  • Stay in Plan mode and wonder why nothing is happening. Plan mode never edits.
  • Leave Auto Accept on overnight. Distracted you is not the same as trusted you.
  • Switch to Auto in a repo you have not committed cleanly first.
  • Treat the modes as a setting you set once. They are a knob. Turn it.
Newbie trap Forgetting which mode you are in. The indicator is small. Glance at it before you walk away from the keyboard.

Exercise. Open elad-test. Press Shift+Tab twice to enter Plan mode. Ask: "propose a plan to add three more journal entries to my journal folder, dated for the next three days." Read the plan. Now Shift+Tab back to Auto Accept Edits and say "go." Notice how the modes split thinking from doing.
