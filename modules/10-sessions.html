Sessions. Many conversations. None of them lost.

A session is one continuous conversation with Claude in one folder. You can run several at once, name them, leave them, come back to them. Learning to manage sessions makes you twice as productive overnight.

The mental model

Each session is its own context window, its own history, its own memory of what was said. Sessions are tied to a folder. Open the same folder later and you can pick up where you left off. Open a different folder and you get a clean session for that one.

The four moves

/rename my-task — give the current session a name so future-you can find it.

/clear — end the current session and start fresh in the same folder. The old one is saved.

/resume — open a picker of past sessions in this folder. Click one to continue it.

/branch — fork the current session. Try an alternative approach without losing the original.

Parallel sessions, the right way

You can open several Desktop windows on the same machine, each with a different project folder. Each is an independent agent. Refactor in one, write copy in another, debug in a third.

For parallel work in the same repo, use git worktrees (full module next: 12 — Git basics for agents). A worktree is a separate folder for the same repository on a different branch. Open Claude Code in each. They cannot collide.

Best practices

Do

  • Rename any session you might want to find again. /rename auth-refactor.
  • /clear when you switch from one task to a totally unrelated one in the same folder.
  • Use git worktrees for true parallel work in one repo. One Claude per worktree.
  • Use /branch when you want to try "what if I did it the other way" without losing your current thread.
  • Treat /resume as your save game menu. It is.

Don't

  • Run two Desktop sessions on the exact same folder at the same time. They will trip over each other.
  • Let one session run forever. Past 1,000 turns, start a fresh one and link it via CLAUDE.md.
  • Delete ~/.claude/projects/. That is where your session history lives.
  • Forget you have a session running in the other window. Glance at the activity indicator.
  • Use /branch when you actually need a worktree. Branches are conversational forks; worktrees are filesystem forks.
Newbie trap Assuming each tab in the Desktop app is a separate session. Multiple tabs of the same project share state. Multiple windows on different folders are independent. If unsure, use a fresh window.

Exercise. In elad-test, run /rename day-one. Send a message: "remember the name of my favorite color is teal." /clear. New session. Ask "what is my favorite color?" — it will not know. Now /resume day-one. Ask again. It remembers.
