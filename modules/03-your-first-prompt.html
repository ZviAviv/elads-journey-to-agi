Part 2 · The Engine 03

Your first prompt. Stop reading. Start using.

You will learn more from one real session than from finishing this guide. So: install the app, sign in, write a prompt that actually moves something. Then come back.

Get the app installed

  1. Go to claude.com/download. Pick Mac or Windows.
  2. Install like any other app. Open it. Sign in with your Anthropic account.
  3. You will land on three tabs at the top: Chat, Cowork, Code. Click Code.
  4. It will ask for a folder. Make a new empty folder somewhere safe (Desktop is fine), call it elad-test, and select it. This is your first project.

The prompt

Type this into the message box at the bottom and send it:

Make me a single HTML file called hello.html that says
"Hello, Elad" in big black letters on a white page. Use one sans-serif font.
Open it in my default browser when you are done.

Watch what happens. Claude will tell you the plan, ask permission to write the file, write it, and then run a command to open it. You should see a browser tab open with your name in it. That is your first agentic loop.

What just happened

You gave Claude a goal in plain English. Claude turned it into a sequence: draft the file, write to disk, open in browser. Each tool use asked you to confirm. You said yes. The loop closed.

Now do it again, slightly different. Try: "change Hello to Welcome and the background to off-white." See how the agent reads your existing file before editing? That is what makes this different from chat.

A prompt is a brief to a junior teammate, not a search query. State the goal, hand over the context, name what "done" looks like. — The whole guide in one sentence

Six rules for prompts that work

1. State the outcome, not the steps.

Bad: "open vscode and create a file and write...". Good: "create hello.html that shows my name." Let the agent figure out how.

2. Give context the agent cannot guess.

Bad: "fix the bug." Good: "the page in hello.html shows blank in Safari but works in Chrome. Find why and fix it."

3. Ask for a plan first on anything non-trivial.

Type: "plan first, do not edit anything yet." Claude will think out loud and propose steps. You read, push back, then say go. Saves hours.

4. Name files and folders explicitly.

Use real paths: src/auth/login.tsx not "the login file." If you do not know the path, ask Claude to find it first.

5. Define done.

Add success criteria: "done means the test in tests/login.test.js passes and there are no console warnings." Now Claude knows when to stop.

6. Iterate. Do not write the perfect prompt.

You are not engineering a one-shot wonder. You are conversing. Send a 70%-good prompt, react to what comes back, steer. Conversation beats manuscript every time.

Best practices

Do

  • Lead with the goal, then constraints, then details.
  • Paste error messages and stack traces verbatim. Never summarize them.
  • Ask for a plan before any change you would not want to undo.
  • Reference patterns: "use the same style as src/components/Button.tsx."
  • Tell the agent what not to do, not just what to do. "Do not change the public API."
  • End with success criteria. One sentence is enough.
  • Show examples or screenshots when the work is visual. Drag the image into the chat.

Don't

  • Write a prompt like a Slack message. Specificity is kindness.
  • Stack ten asks into one prompt. One coherent task per turn.
  • Assume the agent remembers yesterday's session. Restate the spine.
  • Bury the key requirement in the third paragraph. Lead with it.
  • Apologize or pad ("if you don't mind, could you maybe..."). It does not improve results.
  • Send "fix it" without saying what "it" or "fixed" mean.
Newbie trap Trying to write the perfect prompt before you press Enter. Agents work in conversation. Send something rough. React to what comes back. The first turn is a probe, not a contract.

Exercise. Stay in your elad-test folder. Without using a search engine, get Claude to: (1) make a folder called journal; (2) create a markdown file inside it for today's date with a heading and three bullet points about what you learned in this module; (3) tell you the full path to the file you just made.

If anything goes sideways, tell Claude what you wanted instead. That is the loop.
