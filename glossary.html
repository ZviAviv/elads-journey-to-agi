Glossary.

Press / on any page to jump here

  • Agent
    A language model with access to tools (file system, shell, web) that can take actions on its own in a loop, not just produce text. Claude Code is an agent. ChatGPT in your browser is not.
  • Anthropic
    The company that builds Claude. Headquartered in San Francisco, founded in 2021.
  • Ask permissions mode
    The default Claude Code mode. Claude prompts before every file edit and shell command. Safest, slowest.
  • Auto Accept Edits
    A mode where file edits and basic file operations happen automatically. Other shell commands still ask. Switch with Shift+Tab.
  • Auto-memory
    A file Claude maintains in ~/.claude/projects/<project>/memory/ with things it has learned about you. You can read, edit, or delete entries via /memory.
  • Bypass permissions
    A mode that disables all confirmation prompts. Only safe inside sandboxes or VMs. Never run on your main machine.
  • Chat (Claude.ai)
    The browser-based chat interface. Stateless, no file access, good for explanations and brainstorming.
  • Claude Code
    Anthropic's agentic coding tool. Available as a Desktop app (Mac, Windows) and a CLI. The subject of this guide.
  • Claude Design
    An Anthropic Labs product launched April 2026 for visual creation: prototypes, mockups, decks, design systems. Powered by Opus 4.7. See module 16.
  • CLAUDE.md
    A Markdown file at the root of your project containing facts and conventions Claude should know. Loaded into every session in that folder. Generate one with /init.
  • Compaction
    The automatic process of summarizing old conversation when the context window fills, so the session can continue. Trigger manually with /compact focus on ….
  • Context window
    The total amount of text Claude can see in one turn: messages, files, tool outputs, system prompts. Sonnet/Opus default to 200,000 tokens; up to 1M with extended context.
  • Cowork
    Anthropic's surface for non-developer knowledge work. Connects to Drive, Gmail, DocuSign, FactSet. Manipulates documents, no code required.
  • Custom slash command
    A slash command you define yourself by creating a skill in .claude/skills/<name>/SKILL.md. Folder name becomes the command name.
  • Effort
    A dial that controls how hard the model thinks per turn. Set with /effort medium|high|xhigh|max. Higher = more tokens spent reasoning.
  • Extended context
    A 1-million-token context window option, available as opus[1m] or sonnet[1m]. Same per-token cost; useful for huge codebases.
  • Fast mode
    A toggle (/fast) that uses Opus 4.6 instead of 4.7. Slightly less capable, noticeably faster. Only available on Opus.
  • Frontmatter
    YAML metadata at the top of a Markdown file (between --- markers). In skills, defines the name, description, and behavior of the skill.
  • GAI
    Whatever you want it to mean. The point is the journey.
  • Git
    The version control system Claude Code integrates with. Used to track file history, create commits, branches, and worktrees.
  • Git worktree
    A separate folder on your filesystem checked out to a different branch of the same repository. Lets you run two Claude sessions on the same repo without collision.
  • Haiku
    The fastest, cheapest model in the Claude family. Current: Haiku 4.5. Good for quick tasks, formatting, tests.
  • Hook
    A script that runs at a specific moment in the agent loop (PreToolUse, PostToolUse, Stop, etc). Configured in .claude/settings.json. Used for guarantees, not guidance.
  • MCP (Model Context Protocol)
    An open standard for connecting Claude to external tools (GitHub, Slack, Linear, databases). MCP servers expose tools Claude can call.
  • Mode
    How much permission the agent has to act without asking. Five modes: Ask, Auto Accept Edits, Plan, Auto, Bypass. Cycle with Shift+Tab.
  • Model
    Which version of Claude is doing the thinking. Pick with /model. Three families: Opus, Sonnet, Haiku.
  • Opus
    The most capable Claude model. Current: Opus 4.7. Best for hard reasoning, complex refactors, architecture.
  • opusplan
    An alias that uses Opus while planning and switches to Sonnet to execute. Best of both worlds for big tasks.
  • Permissions
    Rules in settings.json that pre-approve, ask, or block specific tool uses. Three buckets: allow, deny, ask. Deny beats allow.
  • Plan mode
    A read-only mode where Claude can look, search, and propose but cannot edit or execute. Approve the plan, then switch modes to act.
  • Plugin
    A bundle of Claude Code config (skills, hooks, subagents, MCP servers) that ships as one installable package. Browse via /plugins.
  • Prompt
    What you type into the message box. For agents, prompts work best when they state the goal, give context, and define done.
  • Session
    One continuous conversation with Claude in one folder. Sessions are saved; resume them with /resume.
  • settings.json
    The config file for permissions, hooks, plugins, and defaults. Three locations: user, project, local.
  • Skill
    A small Markdown file (SKILL.md) that teaches Claude a procedure. Folder name becomes the slash command. Lives in .claude/skills/ or ~/.claude/skills/.
  • Slash command
    A command typed into Claude Code starting with /. Built-in commands like /help, /init, /model, plus any skills you have installed.
  • Sonnet
    The balanced Claude model. Current: Sonnet 4.6. The default for daily work.
  • Subagent
    A separate Claude with its own context, dispatched by the main session to do one task and return a summary. Manage with /agents.
  • Token
    The unit of text Claude counts. Roughly four characters per token. The context window is measured in tokens.
  • Tool use
    When the model invokes a capability — reading a file, running a shell command, fetching a webpage. The defining feature of an agent.
  • Worktree
    See Git worktree.
